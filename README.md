# PyroLert Fire Alert System (Website)

The Pyrolert Website is a **real-time monitoring dashboard** designed to support the Pyrolert smart smoke detection system. It serves as a centralized platform for monitoring environmental sensor data, alert episodes, and occupant headcount detection results generated by the Raspberry Pi.

The platform receives synchronized data through **Supabase Realtime subscriptions**, allowing monitoring personnel to observe environmental conditions and fire-related incidents in real time.

Core features include:

- **Real-time sensor monitoring** for:
  - CO (Carbon Monoxide)
  - NO₂ (Nitrogen Dioxide)
  - O₂ (Oxygen)
  - PM2.5 (Particulate Matter)
  - Temperature
  - Temperature Rate of Change (RoC)
- **Live alert status monitoring** (**Normal, Warning, High Alert**)
- **Historical alert episodes and sensor logs**
- **YOLO-based headcount logs and captured images**
- **Website alarm sounds** for Warning and High Alert events
- **Raspberry Pi inactivity detection** after 30 seconds of no sensor updates
- **Role-Based Access Control (RBAC)** using Supabase Authentication for:
  - Pyrolert Administrator
  - Security Personnel / Facility Management Officer
  - Director / Dean

The website functions as a **remote monitoring platform only**. Smoke detection logic, sensor processing, YOLO inference, and email notifications are handled by the **Raspberry Pi**, while the web platform focuses on **real-time monitoring, visualization, and alert awareness**.

## Development Setup

### 1. Install Dependencies

```bash
npm install
```

### 2. Configure Environment Variables

Create a `.env.local` file and configure the required Supabase credentials:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url_here
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your_supabase_anon_or_publishable_key_here
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key_here

GMAIL_USER=your_gmail_address_here
GMAIL_APP_PASSWORD=your_gmail_app_password_here

NEXT_PUBLIC_PYROLERT_LOGO_URL=https://your-supabase-storage-url-here/pyrolert_light.png

CRON_SECRET=your_secure_random_cron_secret_here
```

### 3. Run Development Server

```bash
npm run dev
```

The application will start locally at:

```txt
http://localhost:3000
```

### 4. Production Deployment

The platform is deployed using **Vercel** and synchronized with **Supabase** for real-time cloud monitoring.
